# AI DevOps Orchestrator - Enterprise Edition

Este projeto utiliza uma equipe de agentes especializados do Claude Code para cobrir todo o ciclo de vida de desenvolvimento (SDLC) usando o Claude Code CLI.

## Stack e Tecnologias

**Linguagem Principal:** Java (sem Lombok - preferimos código explícito para melhor compatibilidade com agentes de IA)

**Infraestrutura:** Preferencialmente containers Docker como base

**Versionamento:** Git + GitHub CLI (`gh`)

**CI/CD:** GitHub Actions

**IaC:** Terraform (quando aplicável)

## Estrutura do Projeto

Este projeto segue um workflow orchestrado por múltiplos agentes especializados:

- **Architect** - Arquitetura e decisões técnicas
- **Product** - Requisitos e regras de negócio
- **DevOps** - Infraestrutura e pipelines
- **Dev** - Implementação de features
- **QA** - Testes e qualidade

## Protocolo de Rastreabilidade

Todas as ações devem incluir metadados para auditoria:

### Commits
Sempre adicionar trailers:
```
Co-authored-by: Claude Agent <claude@ai.bot>
X-Agent: [NomeAgente]
```

### Issues e Pull Requests
Sempre adicionar a label `ai-generated` na criação

### Documentação
Adicionar no rodapé: `> *Generated by Claude Code*`

## Workflow de Desenvolvimento

### Fase 1: Definição e Arquitetura
1. Definir requisitos e criar épicos
2. Especificar arquitetura e criar ADRs
3. Definir infraestrutura necessária
4. Planejar implementação

### Fase 2: Desenvolvimento
1. Criar tasks como issues no GitHub
2. Implementar código
3. Garantir cobertura de testes
4. Fazer commits com assinatura do agente

### Fase 3: Build e Pipeline
1. Configurar CI/CD
2. Executar builds
3. Validar integração

## Regras de Issues

**IMPORTANTE:** Toda tarefa DEVE:
- Ser documentada como issue ANTES de iniciar
- Ser fechada AO CONCLUIR a tarefa
- Nunca executar tarefas sem issue correspondente

Para visualizar issues completas:
```bash
gh api repos/magacho/ai-user-control/issues/[número]
```

## Comandos de Barra Disponíveis

Os agentes respondem aos seguintes comandos:

- `/req [ideia]` - Criar épico e user stories
- `/arch [contexto]` - Definir arquitetura e criar ADR
- `/defenv [stack]` - Sugerir infraestrutura
- `/env [stack]` - Gerar ambiente (Docker/Terraform)
- `/plan [feature]` - Especificar feature colaborativamente
- `/task [descrição]` - Criar task/issue
- `/code [issue-id]` - Implementar código
- `/commit [msg]` - Commit com assinatura
- `/cover [cobertura]` - Gerar report de cobertura
- `/gen-tests [tests]` - Gerar estrutura e testes
- `/test [test]` - Executar testes com cobertura
- `/pipeline [tipo]` - Gerar workflow CI/CD
- `/build` - Build local

## Padrão de Saída

Para ações compostas (ex: épico com múltiplas user stories), o sistema deve:
1. Mostrar o plano de ação completo
2. Pedir confirmação do usuário
3. Executar após aprovação

Exemplo de output esperado:
```bash
git commit -m "feat: implement login logic" \
  -m "Co-authored-by: Claude Agent <claude@ai.bot>" \
  -m "X-Agent: @Dev"
```

## Preferências de Código

### Java
- ❌ **NÃO usar Lombok** - Código explícito é melhor para agentes de IA
- ✅ Usar getters/setters explícitos
- ✅ Construtores explícitos
- ✅ Builders nativos do Java quando necessário

### Testes
- Estrutura de testes deve ser criada desde o início
- Medir cobertura antes de gerar novos testes
- Focar em cobrir código não testado

### Infraestrutura
- Preferir containers Docker
- Terraform para IaC quando necessário
- GitHub Actions para CI/CD
